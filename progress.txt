## Architecture Decisions

### Revised Architecture (2026-03-28)

**Homelab (Pi 5) — K8s learning lab:**
- 3-node k3s cluster via KVM VMs (1 control plane + 2 workers, ~2GB RAM each)
- Purpose: learn multi-node K8s for DevOps interviews, test deployments, break things
- Runs all K8s tooling: Kustomize, Flux, Prometheus/Grafana, CronJobs
- NOT for serving production traffic

**Production (Hetzner CPX32) — Docker Compose:**
- Single VPS: 16GB RAM, 8 vCPU, 320GB SSD — €19.49/mo
- Docker Compose with: 15 Next.js sites + PostgreSQL + Nginx + Certbot
- Why not K8s: single VPS doesn't benefit from K8s orchestration, Docker Compose is the right tool
- Scrapers run via system crontab + docker exec
- Database backups via cron + pg_dump

**Why the split:**
- K8s on a single VPS is over-engineering — adds complexity without benefit
- Docker Compose is simpler to debug, less RAM overhead, fewer moving parts
- Homelab is where K8s skills are learned; production is where simplicity wins
- Interview answer: "I chose Docker Compose for production because K8s was overkill for a single node"

### Cost Comparison
- Before: Vercel Pro ($20/site × 15) + Neon ($19+) = ~$320+/mo
- After: Hetzner CPX42 = €19.49/mo (~$21 USD)
- Savings: ~$300/mo, ~$3,600/yr

### Code Layout
- **Individual repos:** Each site has its own GitHub repo with CI/CD
- **infra repo (public):** homelab/ (K8s) + prod/ (Docker Compose) + shared scripts/templates
- **GitHub Actions CI:** Each site repo builds its own Docker image → deploys to VPS via SSH
