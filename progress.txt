## Architecture Decisions

### Revised Architecture (2026-04-04)

**Homelab (Pi 5) — K8s learning lab:**
- Single-node k3s (expandable to 3-node KVM cluster later)
- Cilium CNI (eBPF networking, Hubble observability)
- MetalLB (LoadBalancer IPs: 192.168.1.200-250)
- Traefik ingress (k3s default)
- Apps: Linkding (bookmarks) + Wallabag (read-it-later) + PostgreSQL + Redis
- Purpose: learn production K8s for DevOps interviews, blog content

**Production (Hetzner VPS) — Docker Compose:**
- Single VPS: Docker Compose with 15 sites + PostgreSQL + Nginx
- Why not K8s: single VPS doesn't benefit from K8s orchestration

**Why the split:**
- K8s on a single VPS is over-engineering
- Homelab is where K8s skills are learned; production is where simplicity wins
- Interview answer: "I chose Docker Compose for production because K8s was overkill for a single node"
